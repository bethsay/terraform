In 01-aws-s3 project, We had configure awscli with the accesskey and secret of iam_admin and set it as the default user.
To run the eks based aws commands, we could append use the flag "--profile tf-eks-admin" everytime.
Or set the AWS_PROFILE as an environment for your current terminal session and forget about using --profile flag for aws commands.
    #export AWS_PROFILE=tf-eks-admin
The AWS_PROFILE environment variable persists for the duration of the terminal session and is availble to all commands and scripts.
Rerun the export command every time you restart the terminal or if you want to run eks commands on another shell session.
Create a minimal cluster with a single command:
    #eksctl create cluster --name my-first-cluster --region us-east-1 --profile tf-eks-admin
Wont work with free-tier account as default type is m5.large
If your aws is still on free-tier this would fail as default type is m5.large while you are allowed to use only t3(small,micro), c7large, m7large types.
This creates A minimal EKS cluster with 1 worker node Using t3.medium instances (cost-effective)
    #eksctl create cluster --name my-first-cluster --region us-east-1 --nodegroup-name standard-workers --node-type t3.small --nodes 1 --nodes-min 1 --nodes-max 3

Enable Basic scanning at a registry level. This will be applied for every repo in default region.
    #aws ecr get-registry-scanning-configuration

Important Notes
EKS control plane costs ~$0.10/hour regardless of cluster size
Worker nodes incur EC2 charges
Fargate charges per vCPU/memory used
Always delete resources when done to avoid ongoing charges
The eksctl approach is the fastest way to get a minimal Kubernetes cluster running. It handles all the complex networking and IAM setup automatically.
